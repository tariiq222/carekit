Command = يغيّر حالة

CreateBookingCommand → handler يكتب على DB وينشر event.
لا يُرجع بيانات — فقط success/fail.

Query = يقرأ فقط

GetBookingsQuery → handler يقرأ من DB ويُرجع data.
لا side effects أبداً.

Event = ما حدث بالفعل

BookingConfirmedEvent → يُنشر بعد Command ينجح.
tasks وnotifications تستمعان له.

Slice = مجلد واحد مكتفٍ

command + handler + dto + spec.
Feature جديدة = مجلد جديد فقط. لا تعديل على شيء موجود.

القاعدة الذهبية: Controller يستقبل HTTP ويرسل Command أو Query فقط. Handler يحتوي كل الـ business logic للـ feature. Domain لا يعرف Prisma. Infrastructure لا تعرف business rules. كل طبقة تتكلم فقط مع التي تحتها — أبداً مع التي فوقها.
