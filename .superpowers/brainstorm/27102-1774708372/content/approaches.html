3 مقاربات لإصلاح الألوان

المشكلة: deriveCssVars() تشتق light vars فقط — الدارك mode يكسر اختيار العميل بـ hardcoded lime.
كل مقاربة تحل المشكلة بطريقة مختلفة.

A

اشتقاق dark vars في deriveCssVars() — موصى به

نضيف dark variants مباشرة في نفس الدالة. تُحقن على html.dark element بدل :root.

primary dark = نفس الـ hue، lightness 70% (مضمون يقرأ على خلفية داكنة)
accent dark = نفس الـ hue، lightness 65%
blob/ring/shadow تشتق تلقائياً من الـ dark primary
✓ كل شيء تلقائي، لا تغيير في globals.css
✓ يعمل مع كل عميل بدون إعداد
~ يتطلب تحويل Hex → HSL → Hex
B

حذف الـ hardcoded dark overrides من globals.css

نحذف .dark { --primary: #9ADB40 } من globals.css ونترك الـ light vars تسود.

الأزرق القوي مثل #354FD8 سيظهر على الدارك mode بنفس القيمة
→ contrast مشكلة: أزرق غامق على خلفية داكنة صعب القراءة
✓ أبسط تنفيذاً
✗ ألوان داكنة ستختفي على dark background
✗ لا يحل المشكلة الجذرية
C

CSS color-mix() — dark vars بدون JS

نستخدم CSS color-mix(in oklch, var(--primary) 70%, white) في globals.css مباشرة.

.dark {
  --primary: color-mix(in oklch, var(--primary-base) 60%, white);
}
✓ لا JS، يعمل حتى قبل hydration
~ يحتاج تسمية --primary-base في deriveCssVars
~ browser support جيد لكن ليس 100%

توصيتي: A
