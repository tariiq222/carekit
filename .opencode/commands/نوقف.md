# /نوقف — Pause Current Task

## What This Command Does

Saves the current task state and stops the active session cleanly.
No continuation. No analysis. No new execution. Write state and stop.

---

## Trigger

User types: `/نوقف`

---

## Behavior — Exactly This, Nothing More

1. Read the current task context from this session
2. Write `.opencode/context/current-task.json` with the fields below
3. Set `status` to `"paused"`
4. Confirm to user that the task is paused
5. Stop

Do NOT:
- Continue working after writing the file
- Re-analyze the task
- Suggest next steps beyond what goes in `next_steps[]`
- Resume automatically in any future session
- Write prose summaries or explanations

---

## Output Format — Write This Exact JSON Structure

File: `.opencode/context/current-task.json`

```json
{
  "task": "<one sentence — what the task is>",
  "status": "paused",
  "current_stage": "<one of: analyze | implement | test | review | qa | completed>",
  "progress": "<one sentence — what has been done so far in this session>",
  "decisions": [
    "<architectural or behavioral decision made during this session — one per item>"
  ],
  "files": [
    "<relative file path of each file created or modified>"
  ],
  "tests_added": [
    "<relative file path of each test file added or updated>"
  ],
  "next_steps": [
    "<the exact next action to take when resuming — be specific, not vague>"
  ],
  "blockers": [
    "<any unresolved question or blocker that must be addressed before continuing>"
  ],
  "runtime": {
    "confidence_score": 0,
    "escalation_recommended": false,
    "escalation_reason": "",
    "token_risk_level": "<low | medium | high>",
    "context_file_count": 0
  }
}
```

### Field Rules

| Field | Rule |
|-------|------|
| `task` | One sentence. No fluff. |
| `status` | Always `"paused"` when /نوقف is used |
| `current_stage` | The stage that was active when paused. Use exact values: `analyze`, `implement`, `test`, `review`, `qa`, `completed` |
| `progress` | One sentence. What is done. Not what was attempted. |
| `decisions` | Only decisions that affect implementation. Skip obvious things. |
| `files` | All files touched. Paths relative to repo root. |
| `tests_added` | Empty array if no tests were added yet. |
| `next_steps` | Specific. "Continue implement step 3 — add DTO validation for CreateBookingDto" not "continue implementation" |
| `blockers` | Unanswered questions only. Empty array if none. |
| `runtime` | Optional object. Present in newly paused tasks. May be absent in older paused tasks — do not fail if missing. |
| `runtime.confidence_score` | From ARCHITECT analysis. 0 if not yet analyzed. |
| `runtime.escalation_recommended` | From ARCHITECT analysis. |
| `runtime.token_risk_level` | `low` / `medium` / `high` based on context size. |

---

## Confirmation Message to User

After writing the file, say exactly:

```
⏸ Task paused.

Stage saved: [current_stage]
Progress: [progress]

Resume with /نكمل in any future session.
```

Nothing else.

---

## Example

User typed `/نوقف` mid-way through implementing a booking fix.

Written file:
```json
{
  "task": "Fix no-show block not enforced on walk-in bookings",
  "status": "paused",
  "current_stage": "implement",
  "progress": "Architect analysis complete. Executor has modified BookingsService.create() to check no-show flag.",
  "decisions": [
    "Walk-in bookings must also check no-show status — they are not exempt",
    "No-show check uses existing isNoShow field on Booking, not a new flag"
  ],
  "files": [
    "backend/src/modules/bookings/bookings.service.ts"
  ],
  "tests_added": [],
  "next_steps": [
    "Write regression test in backend/src/modules/bookings/tests/bookings.service.spec.ts — scenario: walk-in booking blocked when patient has unresolved no-show",
    "Run write-tests pipeline after test is written"
  ],
  "blockers": [],
  "runtime": {
    "confidence_score": 92,
    "escalation_recommended": false,
    "escalation_reason": "",
    "token_risk_level": "low",
    "context_file_count": 3
  }
}
```

Confirmation:
```
⏸ Task paused.

Stage saved: implement
Progress: Architect analysis complete. Executor has modified BookingsService.create() to check no-show flag.

Resume with /نكمل in any future session.
```
