## Recommended next steps

1. **Fix the dashboard 500 immediately (P0 blocker).** The dev server at `:5103` crashes on every route. Root cause: the worktree at `.claude/worktrees/agent-a0082956c1879581a/apps/dashboard/` lacks `.env.local`, which lives only in `apps/dashboard/.env.local` in the main worktree. The Playwright `webServer` block reuses the existing `:5103` server (which runs from the main worktree), but that server itself is returning 500. Identify the crash in the Next.js server logs and fix the missing or broken environment variable first. Without this, 10+ failures are noise caused by a single root cause.

2. **Fix the `notifications.spec.ts` bell selector (Category A, 4 failures).** The `aria-label` on the notification bell button has drifted. Inspect the actual rendered HTML of `components/header.tsx` to find the current aria-label and update the spec's locator. This is the single highest-ROI selector fix — it clears 4 test failures.

3. **Fix auth-redirect behavior (Category B, 3 multi-tenant failures).** After the 500 is resolved, verify that (a) `logout()` correctly clears tokens and the middleware redirects to `/login`, and (b) unauthenticated access to protected routes triggers the Next.js middleware redirect. These may self-heal once the 500 is resolved, but the `middleware.ts` redirect logic should be verified independently.

4. **Fix `notifications.spec.ts` polling test bug (Category E).** Line 224 calls `page.waitForTimeout(60000)` inside a 60-second test — it always exhausts the timeout. Reduce to `waitForTimeout(5000)` or restructure to wait for a DOM change instead of a fixed delay.

5. **Treat the 138 skipped tests as a backlog, not failures.** Most skips are correct: booking wizard sub-tests guard against missing seed data with `test.skip()`, feature-gated routes are explicitly skipped pending premium-plan setup, and header-interactions requires env-var-gated login. The fix task should create a seeding fixture (`beforeAll`) that creates a test org, booking, and employee via the API, enabling these specs to run without skipping. This is medium effort but high coverage value — it would unlock ~40 more tests.
