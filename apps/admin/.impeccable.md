# apps/admin — Design Context

> **Scope lock:** This document governs `apps/admin/` ONLY (the SaaS super-admin control plane on `:5104`). It does NOT govern `apps/dashboard/` (per-tenant clinic dashboard). Do not propagate any decision here into the tenant dashboard, mobile app, or marketing site. The tenant dashboard is Arabic-first / RTL / glassy / branded-per-tenant — different product, different rules.

## Users

**Primary user: the founder/owner (Tariq).** Daily use. Context = high-stakes monitoring + crisis intervention. The user opens this panel when something is wrong — a tenant is in dunning, a payment failed, a customer is escalating, a metric is off — or to do a deliberate, high-leverage action (move a tenant between plans, refund, suspend, impersonate to debug).

This is **not** a daily-driver operations dashboard for a team of clerks. It's a control room for one person who already understands the system. Optimize for: signal density, decisive action, zero ambiguity about what just happened, audit-trail confidence.

Job to be done:
1. See platform health at a glance — is anything bleeding right now?
2. Drill into a specific tenant / invoice / user fast.
3. Take a destructive or money-moving action with full confidence and a clean audit trail.
4. Get out.

## Brand Personality

**Three words: precise · quiet · in-control.**

Voice: a senior operator's terminal. Confident, no marketing language, no hand-holding, no emoji, no "Welcome back!" greetings. The interface assumes the user knows what they're doing. Errors are blunt. Confirmations are minimal. Numbers are exact.

Emotional priority order: **control → confidence → speed → restraint**. Delight is not a goal here; precision is.

## Aesthetic Direction

**Reference**: Vercel dashboard + Stripe dashboard. Generous whitespace, world-class typography hierarchy, content-first (chrome disappears), restrained color, fast.

**Palette**: black-dominant, accented with one bright electric blue. Neutrals tinted very slightly toward the blue hue (chroma ~0.005–0.01) so surfaces and accent feel of-a-piece. Accent is rare by design — it should mean "this is the action / this is the live state," never decoration.

- Light theme: near-white surfaces with deeply tinted neutral text, electric blue for primary action, near-black headings.
- Dark theme: true near-black surfaces (not navy, not charcoal-blue), the same electric blue popping against it, off-white text. Dark is the canonical mode for this app — the user works late and stares at numbers — but light must be equally polished because the founder reviews this on a bright laptop too.
- Theme toggle exposed in the shell. System preference as default.
- Destructive (red) and warning (amber) reserved exclusively for their semantic meaning. Never as accent.

**Typography**: pair a distinctive geometric/neo-grotesque display face with a tabular-friendly body face. Avoid Inter, DM Sans, Plus Jakarta, IBM Plex, Geist (Vercel's own), Space Grotesk — those are the AI-default trap. Reach for something with character: e.g. **Söhne / Söhne Mono** (commercial, but if available), **General Sans**, **Suisse Int'l**, **Neue Haas Grotesk**, **Reckless** (display) + a clean grotesque (body), or a less-used Google Font like **Mona Sans**, **Hubot Sans**, or **Geist's** alternatives like **Manrope** done with restraint. Numerics MUST be tabular (`font-feature-settings: "tnum"`) everywhere a money / count / id appears — this is non-negotiable for a billing-heavy admin.

**Anti-references** (never look like these):
- Hospital / EHR / medical UIs (no soft greens, no caduceus iconography, no rounded-friendly aesthetics).
- The tenant dashboard's RTL/Arabic glassmorphism direction. This app is LTR English-only and must read as a different product.
- Material Design admin templates, generic shadcn-out-of-the-box, Bootstrap admin, SAP, Retool defaults.
- Glassmorphism, frosted blurs, animated gradient blobs, neon-on-dark "cyberpunk dashboard" aesthetic.
- Decorative sparklines, decorative gradient text, decorative side-stripe borders on cards/alerts (`border-left: 4px solid var(--…)` is banned per the impeccable rules).

## Design Principles

1. **Signal over chrome.** Strip every pixel that isn't a number, a label, a status, or an action. No card-wrappers around card-wrappers. Tables sit bare on the page.
2. **One accent, used rarely.** The electric blue means "primary action" or "live/now." If everything is highlighted, nothing is.
3. **Tabular numerics, monospaced ids.** Money, counts, percentages, currency codes, tenant ids, invoice ids, timestamps — all tabular or monospace. The eye must scan columns of numbers without re-anchoring.
4. **Destructive actions look destructive.** Refund, suspend, waive, change-plan, impersonate — these get a distinct visual treatment (confirm sheet, reason-required, the action button styled to make the user pause). Never a casual button.
5. **Keyboard-first navigation.** Command palette (⌘K), `g + letter` page jumps, `j/k` row navigation in tables. The founder should rarely need a mouse.
6. **The audit trail is visible.** Who did what, when, and why is surfaced in-context — not buried on a separate page. Every destructive action shows the resulting `SuperAdminActionLog` row inline after it commits.
7. **No friendly tone.** No "Great job!", no exclamation marks in copy, no "Oops something went wrong." Errors are factual: `"Refund failed: Moyasar returned 402 (insufficient_funds). Tenant invoice unchanged."`
