# CareKit Dashboard — Figma Design Brief

> هذا الملف يحتوي على كل ما تحتاجه كل صفحة من محتوى وبيانات ووظائف.
> لا يتضمن أي توجيهات تصميمية — فقط "ماذا" وليس "كيف".
> المنصة: RTL (عربي أولاً) + LTR (إنجليزي)
> عدد الصفحات: 15 صفحة + صفحة Login

---

## الصفحات

### 1. تسجيل الدخول (Login)

**الحقول:**
- البريد الإلكتروني (email)
- كلمة المرور (password) — مع زر إظهار/إخفاء
- زر "تسجيل الدخول"
- رابط "نسيت كلمة المرور؟"

**حالات:**
- حالة عادية
- حالة تحميل (الزر يدور)
- حالة خطأ (بريد أو كلمة مرور خاطئة)

**عناصر ثابتة:**
- لوقو العيادة (قابل للتغيير — White Label)
- اسم العيادة
- زر تبديل اللغة (عربي/English)
- زر تبديل المظهر (فاتح/داكن)

---

### 2. Layout — الإطار العام (يظهر في كل الصفحات)

#### القائمة الجانبية (Sidebar)
- لوقو العيادة + اسمها
- 3 مجموعات تنقل:

**الرئيسية:**
- لوحة التحكم
- المواعيد
- الممارسون
- المرضى
- الخدمات

**المالية:**
- المدفوعات
- الفواتير
- التقارير

**الإدارة:**
- المستخدمون
- الإشعارات
- المساعد الذكي
- الإعدادات

**حالات العنصر:** عادي، hover، نشط (الصفحة الحالية)
**السايدبار يدعم:** مفتوح (220px) + مطوي (أيقونات فقط) + موبايل (sheet)

#### الهيدر (Header)
- زر طي/فتح القائمة الجانبية
- عنوان الصفحة الحالية
- زر تبديل اللغة
- زر تبديل المظهر (فاتح/داكن/نظام)
- زر الإشعارات + عدد غير المقروءة (رقم أحمر)
- بطاقة المستخدم: صورة/أحرف + الاسم + الدور + dropdown (الملف الشخصي، الإعدادات، تسجيل الخروج)

---

### 3. الصفحة الرئيسية (Dashboard Home)

**4 بطاقات إحصائية (صف أفقي):**

| البطاقة | القيمة | مؤشر التغيير |
|---------|--------|-------------|
| إيرادات اليوم | مبلغ بالريال (مثل: SAR 12,450) | نسبة مئوية ↑ أو ↓ مقارنة بالأمس |
| حجوزات اليوم | رقم (مثل: 45) | نسبة مئوية ↑ أو ↓ |
| إجراءات معلقة | رقم (مثل: 7) | بدون مؤشر |
| مرضى جدد (هذا الأسبوع) | رقم (مثل: 18) | نسبة مئوية ↑ أو ↓ |

كل بطاقة تحتوي: عنوان + رقم كبير + أيقونة + مؤشر التغيير (أخضر للإيجابي، أحمر للسلبي)

**قسمين جنب بعض:**

**يسار — جدول مواعيد اليوم:**
- عنوان "جدول اليوم" + رابط "عرض الكل"
- جدول بأعمدة: المريض (صورة + اسم + ID)، الممارس، الوقت، النوع (شارة ملونة)، الحالة (شارة ملونة)
- يعرض 5 مواعيد

**يمين — الإجراءات المعلقة:**
- عنوان "الإجراءات المعلقة"
- 3 بطاقات:
  - طلبات إلغاء (عدد + زر "عرض الطلبات")
  - تحويلات بنكية معلقة (عدد + زر "مراجعة التحويلات")
  - مواعيد معلقة التأكيد (عدد + زر "عرض الكل")
- زر "عرض كل الإجراءات" أسفل

**قسمين جنب بعض:**

**يسار (عريض) — chart الإيرادات:**
- عنوان "الإيرادات — آخر 30 يوم"
- Line chart — المحور الأفقي: التواريخ، العمودي: المبلغ بالريال

**يمين — chart توزيع الحجوزات:**
- عنوان "توزيع الحجوزات"
- Donut chart — 3 شرائح: زيارة عيادة، استشارة هاتفية، استشارة مرئية
- Legend أسفل الـ chart

**النشاط الأخير:**
- عنوان "النشاط الأخير"
- قائمة عمودية — كل عنصر: أيقونة ملونة حسب النوع + وصف + الوقت النسبي (منذ 5 د، منذ ساعة)
- أنواع النشاط: حجز جديد، تأكيد حجز، إلغاء حجز، دفعة مستلمة، مريض جديد

### 4. المواعيد (Appointments)

**header الصفحة:**
- عنوان "المواعيد" + وصف
- أزرار: تبديل عرض (جدول/تقويم) + تصدير CSV + حجز جديد

**شريط الفلاتر:**
- بحث نصي
- فلتر الحالة: الكل، قيد الانتظار، مؤكد، مكتمل، ملغى، بانتظار الإلغاء، لم يحضر
- فلتر النوع: الكل، زيارة عيادة، استشارة هاتفية، استشارة مرئية
- فلتر الممارس: dropdown بأسماء الممارسين
- فلتر التاريخ: من — إلى

**عرض الجدول:**
- أعمدة: checkbox، المريض (صورة + اسم + ID)، الممارس، النوع (شارة)، التاريخ والوقت، المبلغ، الحالة (شارة)، إجراءات (⋯)
- Pagination أسفل: "عرض 1-10 من 248" + أرقام الصفحات

**عرض التقويم:**
- تقويم شهري
- كل يوم يعرض عدد المواعيد أو نقاط ملونة
- الضغط على يوم يعرض قائمة مواعيد ذلك اليوم

**Panel تفاصيل الموعد (sheet جانبي):**
يفتح عند الضغط على صف في الجدول.
- بيانات الموعد: رقم الحجز، التاريخ، الوقت، النوع، الحالة
- بيانات المريض: اسم + ID + جوال + بريد
- بيانات الممارس: اسم + تخصص
- الخدمة: اسم + سعر
- بيانات الدفع: الطريقة + المبلغ + الحالة
- رابط Zoom (لو استشارة مرئية)
- Timeline: خطوات الموعد (إنشاء → دفع → تأكيد → اكتمال)
- أزرار الإجراءات (حسب الحالة):
  - pending: "تأكيد"
  - confirmed: "إكمال" + "عدم حضور"
  - pending_cancellation: "قبول الإلغاء" + "رفض الإلغاء"
  - أي حالة: "إعادة جدولة"

**Modal قبول الإلغاء:**
- يظهر عند الضغط "قبول الإلغاء"
- حقل "مبلغ الاسترداد" (رقم — كامل أو جزئي أو صفر)
- ملاحظات
- زر تأكيد + إلغاء

**فورم حجز جديد (modal أو صفحة):**
- اختيار المريض (بحث + dropdown)
- اختيار الممارس (dropdown)
- اختيار الخدمة (dropdown — يتغير حسب الممارس المختار)
- نوع الحجز: زيارة عيادة / هاتفية / مرئية (radio)
- التاريخ (date picker)
- الوقت (اختيار من الأوقات المتاحة — grid أزرار)
- ملخص: المبلغ المطلوب
- زر "إنشاء الحجز"

### 5. الممارسون (Practitioners)

**header الصفحة:**
- عنوان "الممارسون"
- زر "إضافة ممارس"

**شريط الفلاتر:**
- بحث نصي
- فلتر التخصص (dropdown)
- فلتر الحالة: نشط / غير نشط

**الجدول:**
- أعمدة: الممارس (صورة + اسم)، التخصص، البريد، الجوال، التقييم (نجوم + رقم)، الحالة (نشط/غير نشط)، إجراءات
- الضغط على صف → صفحة التفاصيل

**فورم إضافة ممارس (modal):**
- الاسم (عربي)
- الاسم (إنجليزي)
- البريد الإلكتروني
- كلمة المرور
- رقم الجوال
- التخصص (dropdown من API)
- السيرة الذاتية (textarea)
- زر "إضافة"

**صفحة تفاصيل الممارس — 5 تبويبات:**

**تبويب 1 — الملف الشخصي:**
- الاسم (عربي + إنجليزي)
- البريد + الجوال
- التخصص
- السيرة الذاتية
- التقييم المتوسط + عدد التقييمات
- حالة الحساب (نشط/غير نشط) — switch للتفعيل/التعطيل
- السعر الافتراضي (لو ما في خدمة معينة)

**تبويب 2 — الخدمات:**
- قائمة الخدمات المعينة للممارس
- كل خدمة: اسم + 3 أسعار (عيادة/هاتف/مرئي) + المدة + حالة (متاح/غير متاح)
- زر "تعيين خدمة جديدة"
- زر حذف لكل خدمة
- فورم تعيين خدمة: اختيار الخدمة (dropdown) + 3 أسعار + المدة + toggle التوفر

**تبويب 3 — الجدول الأسبوعي:**
- جدول 7 أيام (الأحد → السبت)
- كل يوم: toggle تفعيل + وقت البدء + وقت الانتهاء + وقت الراحة
- زر "حفظ الجدول"

**تبويب 4 — الإجازات:**
- قائمة الإجازات: تاريخ البداية + تاريخ النهاية + السبب
- زر "إضافة إجازة" — فورم: تاريخ من + تاريخ إلى + السبب
- زر حذف لكل إجازة

**تبويب 5 — التقييمات:**
- التقييم المتوسط الإجمالي (رقم كبير + نجوم)
- توزيع التقييمات (5 أشرطة: 5 نجوم = 60%، 4 نجوم = 25%، ...)
- قائمة التقييمات: نجوم + تعليق + التاريخ (بدون اسم المريض — مجهول)

### 6. المرضى (Patients)

**header الصفحة:**
- عنوان "المرضى"

**شريط الفلاتر:**
- بحث نصي (اسم أو جوال أو بريد)

**الجدول:**
- أعمدة: المريض (صورة + اسم + ID)، البريد، الجوال، الجنس، تاريخ التسجيل، آخر زيارة، الحالة، إجراءات
- Server-side pagination: "عرض 1-20 من 1,240" + أرقام الصفحات + اختيار عدد الصفوف (10/20/50)

**صفحة تفاصيل المريض — 3 تبويبات:**

**تبويب 1 — الملف الشخصي:**
- الاسم (عربي + إنجليزي)
- البريد + الجوال
- الجنس + تاريخ الميلاد
- تاريخ التسجيل
- حالة الحساب (نشط/غير نشط) — switch
- ملخص النشاط: إجمالي المواعيد + المكتملة + الملغاة + إجمالي المدفوعات

**تبويب 2 — المواعيد:**
- جدول مواعيد المريض: التاريخ، الممارس، الخدمة، النوع، الحالة
- فلتر بالحالة
- Pagination

**تبويب 3 — المدفوعات:**
- جدول المدفوعات: التاريخ، الخدمة، المبلغ، طريقة الدفع، الحالة
- Pie chart: توزيع المدفوعات حسب الحالة
- إجمالي المدفوع

### 7. المدفوعات (Payments)

**4 بطاقات إحصائية:**

| البطاقة | القيمة |
|---------|--------|
| إجمالي المدفوعات | عدد |
| المدفوعة | عدد |
| المعلقة | عدد |
| المستردة | عدد + إجمالي المبلغ المسترد |

**شريط الفلاتر:**
- بحث نصي
- فلتر الحالة: الكل، معلق، مدفوع، مسترد، فشل
- فلتر طريقة الدفع: الكل، Moyasar، تحويل بنكي
- فلتر التاريخ: من — إلى

**الجدول:**
- أعمدة: رقم الدفعة (ID)، المريض، الخدمة، المبلغ، طريقة الدفع (شارة)، الحالة (شارة)، التاريخ، إجراءات
- شارة طريقة الدفع: "إلكتروني" (Moyasar) أو "تحويل بنكي"
- شارة الحالة: معلق (برتقالي)، مدفوع (أخضر)، مسترد (بنفسجي)، فشل (أحمر)
- Pagination

**Modal مراجعة التحويل البنكي:**
يفتح عند الضغط "مراجعة" على تحويل بنكي معلق.
- صورة الإيصال (كاملة، قابلة للتكبير)
- نتيجة تحليل AI:
  - الحالة: matched / amount_differs / suspicious / old_date / unreadable
  - المبلغ المستخرج من الإيصال vs المبلغ المطلوب
  - ملاحظات AI
- حقل ملاحظات الأدمن (textarea)
- زرين: "قبول" (أخضر) + "رفض" (أحمر)

**Modal الاسترداد:**
- المبلغ الأصلي
- حقل مبلغ الاسترداد (رقم — كامل أو جزئي)
- ملاحظات
- زر "تأكيد الاسترداد"

### 8. الفواتير (Invoices)

**4 بطاقات إحصائية:**

| البطاقة | القيمة |
|---------|--------|
| إجمالي الفواتير | عدد |
| المرسلة | عدد |
| غير المرسلة | عدد |
| إجمالي المبلغ | مبلغ بالريال |

**شريط الفلاتر:**
- بحث نصي
- فلتر حالة ZATCA: الكل، QR فقط، بانتظار الإبلاغ، تم الإبلاغ، فشل
- فلتر التاريخ: من — إلى

**الجدول:**
- أعمدة: رقم الفاتورة، المريض، الممارس، المبلغ (شامل الضريبة)، التاريخ، حالة الإرسال (مرسلة/غير مرسلة)، حالة ZATCA (شارة)، إجراءات
- إجراءات: عرض، طباعة، تعليم كمرسلة، إبلاغ ZATCA

**عرض الفاتورة (modal أو صفحة):**
- رقم الفاتورة
- بيانات العيادة (اسم + عنوان + رقم ضريبي)
- بيانات المريض
- بيانات الممارس
- الخدمة + المبلغ قبل الضريبة + نسبة الضريبة + مبلغ الضريبة + الإجمالي
- QR code (ZATCA)
- تاريخ الإصدار
- زر طباعة

### 9. الخدمات (Services)

**header الصفحة:**
- عنوان "الخدمات"
- أزرار: "إضافة فئة" + "إضافة خدمة"

**العرض:**
- فئات قابلة للطي (collapsible)
- كل فئة: اسم الفئة + عدد الخدمات + أزرار (تعديل، حذف)
- داخل كل فئة — جدول الخدمات:
  - أعمدة: اسم الخدمة، سعر العيادة، سعر الهاتف، سعر المرئي، المدة (بالدقائق)، الحالة (نشط/غير نشط)، إجراءات (تعديل، حذف)

**فورم إضافة/تعديل فئة (modal):**
- الاسم (عربي)
- الاسم (إنجليزي)
- الوصف (اختياري)

**فورم إضافة/تعديل خدمة (modal):**
- الفئة (dropdown)
- الاسم (عربي)
- الاسم (إنجليزي)
- الوصف (عربي + إنجليزي)
- سعر زيارة العيادة (بالهللات — يعرض بالريال)
- سعر الاستشارة الهاتفية
- سعر الاستشارة المرئية
- المدة بالدقائق
- toggle نشط/غير نشط

### 10. المستخدمون والأدوار (Users & Roles)

**تبويبين:** المستخدمون | الأدوار

**تبويب المستخدمون:**

شريط الفلاتر: بحث + فلتر الدور (dropdown)
جدول: الاسم (صورة + اسم)، البريد، الجوال، الدور (شارة)، الحالة، تاريخ الإنشاء، إجراءات (تعديل، تفعيل/تعطيل، حذف)

فورم إضافة/تعديل مستخدم (modal):
- الاسم (عربي)
- الاسم (إنجليزي)
- البريد الإلكتروني
- كلمة المرور (عند الإنشاء فقط)
- رقم الجوال
- الدور (dropdown: super_admin, receptionist, accountant, practitioner)
- زر حفظ

**تبويب الأدوار:**

بطاقات الأدوار — كل بطاقة:
- اسم الدور (عربي + إنجليزي)
- الوصف
- عدد المستخدمين بهذا الدور
- شارة "نظامي" لو كان دور أساسي (لا يمكن حذفه)
- زر تعديل الصلاحيات + زر حذف

فورم إنشاء دور (modal):
- الاسم (عربي)
- الاسم (إنجليزي)
- Slug (تلقائي من الاسم الإنجليزي)
- الوصف (اختياري)

مصفوفة الصلاحيات (Permissions Matrix):
- جدول: الصفوف = الوحدات (bookings, payments, users, practitioners, ...)
- الأعمدة = الإجراءات (عرض، إنشاء، تعديل، حذف)
- كل خلية = checkbox
- الأدوار النظامية: 5 (super_admin, receptionist, accountant, practitioner, patient)

### 11. التقارير (Reports)

**فلاتر:**
- نطاق التاريخ: من — إلى
- زر "تطبيق"
- زر "تصدير" (CSV أو Excel)

**قسم الإيرادات:**
- Bar chart: الإيرادات الشهرية (المحور الأفقي: الأشهر، العمودي: المبلغ)
- جدول تفصيلي: الشهر، عدد الحجوزات، إجمالي الإيرادات، متوسط سعر الحجز

**قسم الحجوزات:**
- Pie chart: توزيع حسب الحالة (مؤكد، مكتمل، معلق، ملغى، بانتظار الإلغاء)
- أرقام إحصائية: إجمالي الحجوزات + حسب النوع (عيادة، هاتف، مرئي)

**قسم الممارسين (اختياري):**
- جدول: الممارس، عدد الحجوزات، الإيرادات، التقييم المتوسط

### 12. الإعدادات (Settings) — 7 تبويبات

**تبويب 1 — البراندنج:**
- اسم العيادة (عربي + إنجليزي)
- رفع اللوقو (صورة)
- اللون الأساسي (color picker)
- اللون الثانوي (color picker)
- الخط العربي (dropdown)
- الخط الإنجليزي (dropdown)
- زر حفظ + preview للألوان

**تبويب 2 — التواصل:**
- البريد الإلكتروني
- رقم الهاتف
- العنوان (عربي + إنجليزي)
- المدينة
- الموقع الإلكتروني
- حسابات التواصل الاجتماعي (Twitter, Instagram)
- زر حفظ

**تبويب 3 — الدفع:**
- Moyasar Publishable Key (حقل مخفي)
- Moyasar Secret Key (حقل مخفي)
- Callback URL
- toggle تفعيل التحويل البنكي
- اسم البنك + IBAN + اسم صاحب الحساب
- زر حفظ

**تبويب 4 — الإشعارات:**
- مزود البريد (dropdown: Resend / SendGrid)
- API Key (حقل مخفي)
- البريد المرسل (from email)
- toggle تفعيل FCM
- FCM Server Key (حقل مخفي)
- زر حفظ

**تبويب 5 — سياسة الإلغاء:**
- نص السياسة (عربي) — textarea
- نص السياسة (إنجليزي) — textarea
- زر حفظ

**تبويب 6 — التكاملات:**
3 بطاقات:
- Zoom: Account ID + Client ID + Client Secret (مخفية)
- OpenRouter: API Key (مخفي)
- MinIO: Endpoint + Access Key + Secret Key + Bucket (مخفية)
- كل بطاقة بزر حفظ مستقل

**تبويب 7 — ZATCA:**
- toggle مسجل في ضريبة القيمة المضافة
- الرقم الضريبي (يقبل فقط 15 رقم يبدأ بـ 3)
- نسبة الضريبة (افتراضي 15%)
- البيئة: sandbox أو production (radio)
- CSID (لـ Phase 2)
- المفتاح الخاص (لـ Phase 2)
- زر حفظ

### 13. المساعد الذكي (Chatbot)

**3 تبويبات + صفحة فرعية:**

**تبويب 1 — المحادثات:**
- قائمة المحادثات (يسار): اسم المريض + آخر رسالة + الوقت + حالة (نشطة/منتهية)
- نافذة الدردشة (يمين): رسائل المريض (فقاعة يمين) + رسائل البوت (فقاعة يسار) + رسائل الأدوات (إشعار وسط — "تم حجز موعد")
- لا يوجد حقل إدخال — الأدمن يراقب فقط

**تبويب 2 — التحليلات:**
- 4 بطاقات: إجمالي المحادثات، المحادثات النشطة، الـ Handoffs (تحويل لإنسان)، حسب اللغة
- قائمة "أكثر الأسئلة شيوعاً" مع عدد التكرار

**تبويب 3 — الإعدادات:**
- اسم البوت
- رسالة الترحيب
- اللهجة (dropdown: professional, friendly, casual)
- قدرات البوت — toggles:
  - يستطيع الحجز
  - يستطيع التعديل
  - يستطيع طلب الإلغاء
  - يعرض الأسعار
  - يؤكد قبل الحجز
- كل حقل بزر حفظ مستقل (حفظ فوري)

**صفحة قاعدة المعرفة (صفحة منفصلة):**
- جدول: العنوان، الفئة، تاريخ التعديل، إجراءات (تعديل، حذف)
- زر "إضافة مدخلة" + زر "مزامنة تلقائية" (يسحب الممارسين والخدمات)
- فورم (modal): عنوان + فئة (FAQ, policies, services, practitioners, general) + المحتوى (textarea كبير)

### 14. الإشعارات (Notifications)

**header الصفحة:**
- عنوان "الإشعارات"
- زر "تعليم الكل كمقروء"

**فلاتر:**
- فلتر النوع: الكل، تأكيد موعد، اكتمال موعد، إلغاء، تذكير، دفعة، تقييم، بلاغ
- فلتر الحالة: الكل، مقروء، غير مقروء

**القائمة:**
- كل إشعار: أيقونة ملونة حسب النوع + العنوان + الوصف + الوقت النسبي + نقطة "غير مقروء"
- الضغط على الإشعار → يعلمه كمقروء + يحول للصفحة المعنية (مثل: إشعار موعد → صفحة الموعد)

**أنواع الإشعارات وأيقوناتها:**

| إلغاء موعد (booking_cancelled) | CalendarX | أحمر |
| تذكير (reminder) | Bell | برتقالي |
| دفعة مستلمة (payment_received) | CreditCard | أخضر |
| تقييم جديد (new_rating) | Star | بنفسجي |
| بلاغ مشكلة (problem_report) | AlertTriangle | أحمر |

### 15. حالات مشتركة (تظهر في كل الصفحات)

**حالة التحميل (Loading):**
- Skeleton loaders بنفس شكل المحتوى النهائي
- البطاقات الإحصائية: skeleton مستطيل
- الجداول: 5 صفوف skeleton
- Charts: skeleton مربع

**حالة فارغة (Empty State):**
- أيقونة كبيرة
- عنوان "لا توجد بيانات"
- وصف قصير
- زر إجراء (اختياري — مثل "إضافة أول موعد")

**حالة خطأ (Error):**
- بطاقة بخلفية حمراء فاتحة
- أيقونة تحذير + رسالة الخطأ
- زر "إعادة المحاولة"

**Toast/Snackbar:**
- يظهر أسفل يسار الشاشة (أو يمين في RTL)
- نجاح: أيقونة ✓ + رسالة
- خطأ: أيقونة ✕ + رسالة
- يختفي بعد 4 ثواني

**Modal تأكيد الحذف:**
- أيقونة سلة حمراء
- "هل أنت متأكد من حذف [العنصر]؟"
- "لا يمكن التراجع عن هذا الإجراء"
- زرين: "نعم، احذف" (أحمر) + "إلغاء"
