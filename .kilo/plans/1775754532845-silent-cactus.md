# Plan: Multi-Step Group Session Creation Form

## Goal
Convert the single-page "Add Group Session" form (`/group-sessions/create`) into a multi-step wizard following the existing `booking-wizard` pattern.

---

## Step Breakdown (4 Steps)

| Step | Title (AR) | Title (EN) | Fields |
|------|-----------|------------|--------|
| **1** | المعلومات الأساسية | Basic Info | `nameAr`, `nameEn`, `descriptionAr`, `descriptionEn`, `practitionerId` |
| **2** | إعدادات المشاركة | Participation Settings | `minParticipants`, `maxParticipants`, `pricePerPersonHalalat`, `durationMinutes`, `paymentDeadlineHours` |
| **3** | الجدولة والنشر | Scheduling & Publishing | `schedulingMode`, `startTime` (if fixed_date), `isPublished`, `expiresAt` |
| **4** | المراجعة والتأكيد | Review & Confirm | Read-only summary of all fields + Submit button |

## Architecture Decision

**Approach: react-hook-form stays as single source of truth** (not split state like booking wizard).

The booking wizard uses a custom `useState` machine because it has dynamic flow (service-first vs practitioner-first) and doesn't use react-hook-form until the final step.

For group sessions, the form is linear (always same steps) and already uses react-hook-form. So the wizard steps will share the **same `form` instance** and each step validates its subset of fields using `form.trigger(fieldNames)`.

Benefits:
- No state sync issues
- User can go back and forth without losing data
- Final submit reuses existing `onSubmit` logic unchanged

### Validation Strategy
```ts
// Each step validates only its fields
const stepFields: Record<SessionWizardStep, (keyof CreateGroupSessionFormValues)[]> = {
  1: ['nameAr', 'nameEn', 'descriptionAr', 'descriptionEn', 'practitionerId'],
  2: ['minParticipants', 'maxParticipants', 'pricePerPersonHalalat', 'durationMinutes', 'paymentDeadlineHours'],
  3: ['schedulingMode', 'startTime', 'isPublished', 'expiresAt'],
  4: [], // review — no new fields, final submit validates full schema
}
```
