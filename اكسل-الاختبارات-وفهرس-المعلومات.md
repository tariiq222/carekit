## 🗂️ بنية كل ملف إكسل

كل ملف يتبع نفس القالب، ورقة **E2E Test Cases** فيها الأعمدة:

| العمود | الوصف |
|--------|-------|
| Test ID | معرّف فريد (مثال: CL-001 للعميل) |
| **Module** | اسم الموديول (Clients, Bookings, Payments...) |
| Test Type | Positive / Negative |
| Test Scenario | عنوان مختصر للسيناريو |
| Test Case Title | وصف مفصل للحالة |
| Preconditions | الشروط المسبقة قبل التشغيل |
| Test Steps | خطوات التنفيذ |
| Test Data | بيانات الإدخال |
| Expected Result | النتيجة المتوقعة |
| Actual Result | (يُملأ عند التنفيذ) |
| Priority | P1-High / P2-Medium / P3-Low |
| Status | Not Started / Pass / Fail / Blocked |
| Environment | Staging / Production / Local |
| Executed By | منفّذ الاختبار |
| Execution Date | تاريخ التنفيذ |
| Notes / Defect ID | ملاحظات أو رقم الخلل |

ورقة **Summary Dashboard** فيها عدّادات (Total / Pass / Fail / Pending) تعتمد صيغ COUNTIF تلقائية.

## 📊 موديول العملاء — تغطية مختصرة

**المصادر المُفحوصة:**
- Backend: [apps/backend/src/modules/people/clients/](apps/backend/src/modules/people/clients/)
- Controllers: [apps/backend/src/api/dashboard/people.controller.ts](apps/backend/src/api/dashboard/people.controller.ts) + `api/mobile/client/profile.controller.ts` + `api/mobile/employee/clients.controller.ts`
- Schema: [apps/backend/prisma/schema/people.prisma](apps/backend/prisma/schema/people.prisma)
- Dashboard UI: [apps/dashboard/app/(dashboard)/clients/](apps/dashboard/app/(dashboard)/clients/)
- Mobile: [apps/mobile/app/(client)/(tabs)/profile.tsx](apps/mobile/app/(client)/(tabs)/profile.tsx)

**الفئات المغطاة (85 سيناريو):**

| الفئة | إيجابي | سلبي |
|-------|--------|------|
| Create (إنشاء) | 3 | 18 |
| Update (تعديل) | 5 | 6 |
| Delete (حذف) | 2 | 5 |
| Get by ID (عرض التفاصيل) | 3 | 2 |
| List / Search / Filter (القائمة والبحث) | 11 | 5 |
| Mobile Profile | 3 | 3 |
| RTL / UI / Network | 5 | 4 |
| Ratings / History (علاقات) | 3 | 2 |
| Concurrency / Security | 0 | 4 |

**القواعد الرئيسية:**
- Phone regex: `/^\+9665\d{8}$/` (سعودي فقط)
- unique `(tenantId, phone)` — soft-delete يُفرّغ الجوال لإعادة الاستخدام
- Soft delete: `deletedAt` + `isActive=false` + `phone=null` + الجوال القديم يُضاف للـ notes
- Tenant isolation: كل استعلام يفلتر `tenantId` و `deletedAt: null`
- Mobile: `clientId` مجبور من JWT (`user.sub`)

## ➕ إضافة موديول جديد

1. أنشئ سكريبت `scripts/generate_<module>_e2e.py` بنفس بنية سكريبت العملاء.
2. استخدم القالب `e2e_test_template.xlsx` نقطة بداية.
3. ابدأ بترقيم `<PREFIX>-001`.
4. أضف صفاً جديداً في جدول **روابط ملفات الإكسل** أعلاه.
5. وثّق المصادر المُفحوصة (Backend, Frontend, Schema).
