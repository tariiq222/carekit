## 🗂️ بنية كل ملف إكسل

كل ملف يتبع نفس القالب، ورقة **E2E Test Cases** فيها الأعمدة:

| العمود | الوصف |
|--------|-------|
| Test ID | معرّف فريد (مثال: CL-001 للعميل) |
| **Module** | اسم الموديول (Clients, Bookings, Payments...) |
| Test Type | Positive / Negative |
| Test Scenario | عنوان مختصر للسيناريو |
| Test Case Title | وصف مفصل للحالة |
| Preconditions | الشروط المسبقة قبل التشغيل |
| Test Steps | خطوات التنفيذ |
| Test Data | بيانات الإدخال |
| Expected Result | النتيجة المتوقعة |
| Actual Result | (يُملأ عند التنفيذ) |
| Priority | P1-High / P2-Medium / P3-Low |
| Status | Not Started / Pass / Fail / Blocked |
| Environment | Staging / Production / Local |
| Executed By | منفّذ الاختبار |
| Execution Date | تاريخ التنفيذ |
| Notes / Defect ID | ملاحظات أو رقم الخلل |

ورقة **Summary Dashboard** فيها عدّادات (Total / Pass / Fail / Pending) تعتمد صيغ COUNTIF تلقائية.

**الفئات المغطاة (85 سيناريو):**

**القواعد الرئيسية:**
- Phone regex: `/^\+9665\d{8}$/` (سعودي فقط)
- unique `(tenantId, phone)` — soft-delete يُفرّغ الجوال لإعادة الاستخدام
- Soft delete: `deletedAt` + `isActive=false` + `phone=null` + الجوال القديم يُضاف للـ notes
- Tenant isolation: كل استعلام يفلتر `tenantId` و `deletedAt: null`
- Mobile: `clientId` مجبور من JWT (`user.sub`)

## 🧪 تشغيل الاختبارات (Backend Jest E2E)

### متطلبات البيئة المحلية (بدون Docker)

| الخدمة | العنوان | ملاحظات |
|--------|---------|---------|
| Postgres 18 | `127.0.0.1:5432` | مستخدم `carekit` / قاعدة `carekit_test` (أُنشئت بالسوبر-مستخدم `postgres`) |
| Redis 8.x (scoop) | `127.0.0.1:5380` | **ليس** Redis القديم في `C:\Program Files\Redis` (3.0 — BullMQ يرفضه) |

### تشغيل Redis
```bash
"C:/Users/tarii/scoop/shims/redis-server.exe" --port 5380 --save "" --appendonly no
```

### أوامر التشغيل

#### Backend Jest E2E
```bash
# من apps/backend:
TEST_DATABASE_URL="postgresql://carekit:carekit_dev_password@127.0.0.1:5432/carekit_test?schema=public" \
REDIS_PORT=5380 \
npx jest --config test/jest-e2e.json test/e2e/people/clients.e2e-spec.ts \
  --json --outputFile=test-results-clients.json
```

#### Playwright UI (يتطلب backend + dashboard مشغّلَين)
```bash
# تيرمينال 1 — backend على :5100:
cd apps/backend && \
DATABASE_URL="postgresql://carekit:carekit_dev_password@127.0.0.1:5432/carekit_dev?schema=public" \
REDIS_HOST=127.0.0.1 REDIS_PORT=5380 PORT=5100 npm run dev

# تيرمينال 2 — dashboard على :5103:
cd apps/dashboard && npm run dev

# تيرمينال 3 — seed admin (مرة واحدة، إن لم يُنفّذ بعد):
cd apps/backend && \
DATABASE_URL="postgresql://carekit:carekit_dev_password@127.0.0.1:5432/carekit_dev?schema=public" \
npx tsx prisma/seed.ts

# تيرمينال 4 — Playwright:
cd apps/dashboard && \
npx playwright test test/e2e/clients/ --reporter=json 2>/dev/null > playwright-clients-results.json
```

#### دمج النتائج إلى الإكسل
```bash
# من الجذر — يقرأ Jest + Playwright JSON ويحدّث الإكسل:
py scripts/update_xlsx_merged.py
```

#### توليد تقرير HTML تفاعلي 🆕
```bash
# من الجذر — يولّد تقريراً مرئياً standalone من نفس JSONs + الإكسل:
py test-reports/scripts/generate_html_report.py

# فتح التقرير:
start test-reports/output/test-report.html   # Windows
```

**المميزات:** فلترة/بحث ديناميكي، تفاصيل قابلة للطيّ، تنسيق RTL عربي، dark/light تلقائي، ملف HTML واحد مستقل.
راجع [test-reports/README.md](test-reports/README.md) للتفاصيل.
